# 1.3 Recuperación de contraseña — Backend & Frontend

## Goal
Implement forgot-password and reset-password flow. User enters email → receives link → sets new password.

## Backend Changes

### `src/users/user.entity.ts`
- Added `resetPasswordToken: varchar nullable select:false`
- Added `resetPasswordExpires: datetime nullable select:false`

### `src/auth/auth.types.ts`
- Added `ForgotPasswordInput` DTO: `email` (IsEmail, IsNotEmpty)
- Added `ResetPasswordInput` DTO: `token` (IsNotEmpty), `password` (MinLength(8), IsNotEmpty)

### `src/auth/auth.service.ts`
- **`forgotPassword(dto)`**: Finds user by email. If exists → generates `randomUUID()` token + 1h expiry, saves, sends email with link `http://localhost:4200/reset-password?token=<token>`. Always returns `{ message: "Si el email existe, recibirás un link" }` (security: don't reveal if email exists).
- **`resetPassword(dto)`**: Finds user by `resetPasswordToken` (uses `addSelect` because field has `select: false`). Validates not expired. Hashes new password with bcrypt, updates user, clears reset fields. Returns `{ message: "Contraseña actualizada" }` or throws `BadRequestException('Token inválido o expirado')`.

### `src/auth/auth.controller.ts`
- `POST /auth/forgot-password` (no auth) → `authService.forgotPassword(dto)`
- `POST /auth/reset-password` (no auth) → `authService.resetPassword(dto)`

## Frontend Changes

### `services/auth.service.ts`
- Added `forgotPassword(email)` → `POST /auth/forgot-password`
- Added `resetPassword(token, password)` → `POST /auth/reset-password`

### Pages created
- **`forgot-password/`**: Card with email input + "Enviar link" button. On submit → shows generic success message "Si el email existe, recibirás un link" + link back to login. The `catch` also sets `sent = true` to always show the same message (security).
- **`reset-password/`**: Reads `?token=` from queryParams. Shows error if no token. Otherwise shows new password + confirm password form (min 8 chars, must match). On success → "Contraseña actualizada" + link to login. On error → shows error message + link to login.

### `app.routes.ts`
- Added `/forgot-password` → lazy load `ForgotPasswordPage` (no guard)
- Added `/reset-password` → lazy load `ResetPasswordPage` (no guard)

### `login.html`
- Added "¿Olvidaste tu contraseña?" link above the register link, pointing to `/forgot-password`

## Build Verification
- Backend: `nest build` succeeds
- Frontend: `ng build` succeeds, lazy chunks `forgot-password` and `reset-password` appear
