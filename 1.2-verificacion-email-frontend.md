# 1.2 Verificación de email — Frontend

## Resumen

Se implementó el flujo completo de verificación de email en el frontend Angular: pantalla de "revisá tu email" post-registro, página que procesa el token de verificación, y badge de estado en el perfil con opción de reenviar.

## Archivos modificados

### 1. `src/app/services/auth.service.ts` — Líneas 44-50

Se agregaron dos métodos al `AuthService`:

```typescript
verifyEmail(token: string): Observable<{ message: string }> {
  return this.http.post<{ message: string }>(`${this.api}/verify-email`, { token });
}

resendVerification(): Observable<{ message: string }> {
  return this.http.post<{ message: string }>(`${this.api}/resend-verification`, {});
}
```

- `verifyEmail(token)` → `POST /auth/verify-email` con `{ token }` en el body. No requiere JWT (sin guard).
- `resendVerification()` → `POST /auth/resend-verification`. Requiere JWT (se envía automáticamente por el `jwtInterceptor`).

## Archivos creados

### 6. `src/app/pages/verify-pending/verify-pending.ts`

```typescript
export class VerifyPendingPage {
  private auth = inject(AuthService);
  sent = signal(false);
  sending = signal(false);

  async resend(): Promise<void> {
    this.sending.set(true);
    try {
      await firstValueFrom(this.auth.resendVerification());
      this.sent.set(true);
    } catch {
      alert('Error al reenviar el email. Iniciá sesión e intentá de nuevo.');
    } finally {
      this.sending.set(false);
    }
  }
}
```

- `sent`: señal que controla si se muestra el alert de éxito
- `sending`: señal para deshabilitar el botón y mostrar spinner
- El catch captura el error (ej. si no hay JWT) y muestra un alert

### 8. `src/app/pages/verify-email/verify-email.ts`

```typescript
export class VerifyEmailPage implements OnInit {
  private route = inject(ActivatedRoute);
  private auth = inject(AuthService);
  success = signal(false);
  error = signal('');

  async ngOnInit(): Promise<void> {
    const token = this.route.snapshot.queryParams['token'];
    if (!token) {
      this.error.set('Token no encontrado en la URL.');
      return;
    }
    try {
      await firstValueFrom(this.auth.verifyEmail(token));
      this.success.set(true);
    } catch (err: any) {
      this.error.set(err.error?.message || 'Token inválido o expirado.');
    }
  }
}
```

- Implementa `OnInit` para leer el token de los queryParams ni bien se monta el componente
- `ActivatedRoute.snapshot.queryParams['token']` obtiene el token de la URL
- Si no hay token → error
- Si `verifyEmail` falla → muestra el mensaje de error del backend o uno genérico

## Flujo completo

```
Register ──éxito──> /verify-pending
                        │
                        ├── botón "Reenviar email" → POST /auth/resend-verification (JWT)
                        │
                        └── usuario revisa email ──cliquea link──> /verify-email?token=xxx
                                                                       │
                                                                       ├── POST /auth/verify-email { token }
                                                                       │
                                                                       ├── éxito → "Email verificado" → link a /login
                                                                       │
                                                                       └── error → "Token inválido" → link a /login


Profile (autenticado)
  ├── badge "Verificado" (verde) si isVerified
  └── badge "No verificado" (amarillo) + botón "Reenviar email" si !isVerified
```
